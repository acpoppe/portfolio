Latest Projects

Metroid Room Counter and Coordinates Romhack

Metroid was an early NES game which focused on exploration of an alien world. I hacked the rom and modified its code to hack a room tracker into the game loop to show a count of visited rooms in one version and in another to also show the current room coordinates the player occupies. This was useful for practicing challenge runs of the game, like 100% map completion, and being sure that no location was missed. This code won’t be posted publicly due to the copyright nature of the materials, but examples and explanations can be furnished on request. Entirely written in 6502 Assembly and working within the hardware restrictions of a traditional NES cart and the mappers available at the time (If it were put on a cart, it would run on any production NES or emulator).

Ratbit

Again, there’s no publicly available code or release of this one due to possible copyright concerns, but once upon a time, friends and I would enjoy screensharing our browsers or computer screens to watch movies together in a synchronized way. Well, Rabbit never seemed to figure out how to monetize and the service went away (I believe it’s been bought since then, but it’ll never be the same). So, what else could I have done, but write my own version? I called it Ratbit, in honor of my favorite pet and the inspiration project. It features a server side component built on WebRTC components and including features like password protected private rooms and a Google Chrome extension for sharing.

Noobs Ask Noobs Bot

During a discussion in the We Like Dota Community server an idea came up for a bot to handle questions for the podcast so they wouldn’t be lost in the shuffle. We Like Dota is a podcast about Dota, a MOBA from Valve. In it, they regularly have a segment called Noobs Ask Noobs in which they answer questions from their listeners, predominantly asked by those in the server. In a few hours, I had created and hosted a bot for them (although they did request that I name the bot “Alliebot”) and it has been used for a few months daily by users of that server to submit questions, which the hosts can view and respond to before clearing the queue of questions. The bot was written in python using the discord.py library with a PHP and MySQL backend.

Inventory

Inventory is an iOS/Android app designed to make a convoluted restaurant inventory process easier. It allows the user to accurately count items with multiple units of measurement and easily converts them to the one they need to report. For instance if they have 2 cases, 3 sleeves, and 21 individual cups, it would let them know they had 2.16 cases of cups. Lists are also sortable by the location of the item in the business as well as by the report order so the user can walk through the store with everything in the same order and when they sit down to copy their results do the same. It hasn’t been updated in quite a while and likely wouldn’t run on modern versions of either OS, but the code still exists and could be produced to show.
